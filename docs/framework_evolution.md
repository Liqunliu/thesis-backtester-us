# 投资框架自动进化机制

## 一、核心思路

当前流程是**单向的**：人写框架 → AI执行 → 验证。

进化版流程是**闭环的**：

```
框架V1 → AI盲测 → 验证（对/错归因）
                         ↓
                    AI分析错误模式
                         ↓
                    自动生成改进建议
                         ↓
                    框架V2 → AI盲测 → 验证 → ...
```

这不是简单的参数调优，而是让AI基于回测结果**反思自己的分析方法**，识别系统性盲区，提出框架层面的改进。

## 二、已有的错误模式分析

从V5.5.6的60样本盲测中，已识别以下系统性错误：

### 2.1 假阴性（回避但实际大涨）— 9只，占比15%

| 样本 | AI评分 | 实际收益 | 错误原因 |
|------|--------|---------|---------|
| 苏宁环球 2020-12 | 37 | +136.4% | 2021年初牛市，流动性驱动 |
| 物产中大 2020-12 | 37 | +94.5% | 供应链公司被误判为高杠杆 |
| 滨江集团 2021-12 | 38 | +74.7% | 优质民营房企被行业性否定 |
| 鲁西化工 2020-06 | 25 | +73.3% | 化工周期底部，AI无法预判反转 |
| 中远海控 2023-12 | 52 | +61.7% | 航运周期反弹 |
| 居然智家 2024-06 | 53 | +60.1% | 消费复苏+估值修复 |
| 上海建工 2021-06 | 42 | +29.7% | 建筑央企被高杠杆否定 |
| 保利发展 2021-06 | 42 | +27.0% | 优质央企地产被行业性否定 |
| 中国建筑 2023-12 | 41 | +10.4% | 同上 |

**共性特征**：
1. **周期底部反转**：6/9只是周期股在底部被回避
2. **流动性牛市**：3/9只集中在2020-12截面
3. **行业"一刀切"**：3/9只是优质央企地产/建筑被行业整体否定

**根因**：框架缺乏宏观环境维度 + 对周期股用了错误的估值方法 + 对央企地产/建筑的行业歧视过度。

### 2.2 假阳性（高分/买入但实际下跌）— 9只，占比15%

**共性特征**：
1. **周期顶部高估**：7/9只是周期股在景气高点
2. **煤炭景气拐点**：3只煤炭股集中在2024年

**根因**：框架没有"周期位置"判断维度 + 对历史高盈利的外推过于乐观。

### 2.3 观望区无效（25只，均收益仅+0.1%）

"观望"占比41.7%，说明框架在中间地带缺乏区分力。

## 三、框架改进方向

### 3.1 v6 已实施的改进

以下改进已在 v6 框架中通过**算子级别**实施：

| 改进方向 | v6 实现 |
|---------|---------|
| 行业门控 | `valuation_fcf`, `owner_earnings`, `cigar_butt`, `light_asset_model` 均含前置排除门控 |
| 流派分类 | `stream_classification` 算子将标的分为纯股息/价值发现/烟蒂/关联方资源 |
| 央国企识别 | `soe_identification` 算子在 Ch02 前置执行 |
| PE陷阱检测 | `pe_trap_detection` 新增算子 |
| 业绩修复判断 | `performance_restoration` 新增算子 |
| 估值修复催化 | `valuation_repair` 新增算子 |

### 3.2 待实施：新增"宏观环境评估"维度

**目标**：解决假阴性中的"周期反转盲区"和假阳性中的"周期顶部高估"。

**方案**：创建新算子 `macro_environment.md`，在 system prompt 中注入截面时点的宏观环境摘要。

### 3.3 待实施：增强"周期股特殊处理"逻辑

**当前状态**：`cycle_analysis` 算子已存在，但缺乏毛利率趋势拐点识别。

**改进**：扩展 `cycle_analysis` 算子内容，增加：
- 正常化利润计算（5年平均）
- 毛利率趋势方向判断
- PB 估值优先于 PE 估值的规则

### 3.4 待实施：细化"观望区"评分标准

**方案**：在 `strategy.yaml` 的 `synthesis_fields` 中增加强制升降级规则，或创建新的 `forced_override.md` 算子。

## 四、自动进化流程设计

### 4.1 v6 架构下的进化流程

v6 的算子驱动架构使框架进化更加模块化：

```
Step 1: 收集错误案例
  - quality_scorer.py 5维评分 → 识别假阳性/假阴性
  - crosssection.py 跨截面对比 → 识别系统性偏差

Step 2: AI错误归因
  - 输入：agent_reports/*.md + outcome 数据
  - 分析每个错误案例对应的算子输出

Step 3: 算子级修改
  - 修改算子 .md 文件（增加门控/调整权重/扩展分析维度）
  - 新增算子 → 在 strategy.yaml 中引用
  - 无需修改引擎代码

Step 4: 重跑回测验证
  - batch_backtest.py 在相同截面重跑
  - quality_scorer.py 对比前后版本

Step 5: 记录进化历史
  - 每个策略版本保存为独立目录（如 strategies/v7_value/）
```

### 4.2 过拟合防护

1. **训练/验证集分离**：用2019-2022数据做框架优化，用2023-2024做独立验证
2. **逻辑审核**：每个算子修改必须有可解释的投资逻辑
3. **复杂度惩罚**：算子总数不超过 25 个，章节不超过 8 章
4. **版本对比**：每次迭代保存完整的 strategy.yaml，一旦样本外退化即回退

### 4.3 进化指标

| 指标 | 目标方向 | V5.5.6基线 | v6 目标 |
|------|---------|-----------|---------|
| 方向正确率 | ↑ | 42% | 50%+ |
| 策略增益（alpha） | ↑ | +16.6pp | +20pp+ |
| 买入信号胜率 | ↑ | 67% | 75%+ |
| 假阴性率 | ↓ | 15% | <10% |
| 假阳性率 | ↓ | 15% | <10% |
| 观望占比 | ↓ | 42% | <30% |
| 算子数 | → | 14 | 21（当前） |

## 五、与数据扩展的协同

v6 的 `StockSnapshot` 已大幅扩展数据维度（行业分类、审计意见、主营构成、股东户数、股权质押等），为框架进化提供了更丰富的信息基础。

**最优实施顺序**：
1. 数据已扩展 ✓（snapshot.py 含 15+ 数据类型）
2. 算子已增强 ✓（21 算子 + 行业门控）
3. 下一步：增加宏观环境算子 + 全量600样本回测验证
