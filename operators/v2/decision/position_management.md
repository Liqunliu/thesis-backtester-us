## 持仓与仓位管理框架

核心-卫星仓位配置 + 试探建仓网格加仓 + 动态退出纪律 + 持仓状态标签 + 组合韧性评估的完整持仓管理体系。

### 一、核心-卫星仓位配置

基于FCF倍数的仓位上限硬约束：

| 类型 | 定义 | 仓位上限 | 核心特征 | 进入标准 |
|------|------|---------|---------|---------|
| **核心标的** | 长期持有的压舱石 | 15% | 剔除净现金FCF倍数<3倍，无有息负债 | 央国企背景 + 极端低估 |
| **卫星标的** | 阶段性配置标的 | 8% | 剔除净现金FCF倍数3-5倍，事件驱动 | 估值合理 + 有催化剂 |
| **观察标的** | 跟踪研究中 | 2% | 符合标准但估值未到买点 | 插眼试探仓位 |

### 二、试探建仓与网格加仓（6档梯度）

适用场景：标的符合估值标准（FCF倍数<5倍），但价格位置不佳（距52周低点>30%），需验证底部或等待更深回调。

**6档网格加仓框架**：

| 档位 | 触发条件 | 单次仓位 | 累计仓位 | 说明 |
|------|---------|---------|---------|------|
| **第1档：插眼试探** | 首次触及买点区间 | 10% | 10% | 极小仓位验证判断 |
| **第2档** | 价格下跌3% | 15% | 25% | 确认下跌趋势 |
| **第3档** | 价格下跌5% | 15% | 40% | 逐步加大 |
| **第4档** | 价格下跌8% | 20% | 60% | 进入重仓区间 |
| **第5档** | 价格下跌12% | 20% | 80% | 接近满仓 |
| **第6档** | 价格下跌15% | 20% | 100% | 达到目标仓位 |

**加仓触发条件（必须同时满足）**：

```
1. 价格下跌达到触发阈值（3%/5%/8%/12%/15%）
2. 基本面未恶化（FCF/利润未大幅下滑）
3. 估值仍然合理（FCF倍数仍在目标区间内）
4. 总加仓次数不超过6次
```

**加仓禁止事项**：

```
- 单次加仓间隔<3%（避免接飞刀）
- 未达触发价提前加仓（情绪化交易）
- 基本面恶化后继续加仓（价值陷阱）
- 单标的总仓位超过上限（核心15%/卫星8%）
```

**加仓终止条件**：

```
- 估值修复完成（FCF倍数>8倍）→ 停止加仓，持有观望
- 基本面恶化（FCF下滑>30%）→ 停止加仓，评估退出
- 发现更优标的（FCF倍数差>2倍）→ 资金切换
```

### 三、动态退出纪律（4类触发）

#### 3.1 估值修复退出

```
买入逻辑：在FCF倍数X倍买入，目标Y年回报率Z%
退出触发：涨幅透支5-6年股息（估值修复完成）
操作：盈利达目标后分批退出
原则：不赚最后一个铜板
```

#### 3.2 周期股退出

| 触发信号 | 操作 |
|----------|------|
| 周期价格涨至70%分位 | 减仓50% |
| 周期价格涨至90%分位 | 清仓 |
| 经营现金流增速<10% | 减仓至观察仓位 |

#### 3.3 事件驱动退出

| 触发信号 | 操作 |
|----------|------|
| 地缘政治风险暴露 | 即时退出 |
| 政策风险确认 | 减仓至2%以下 |
| 基本面恶化（FCF下滑>30%） | 即时止损 |

#### 3.4 高低切换再平衡

| 触发条件 | 阈值 | 操作 |
|---------|------|------|
| 股息率下降 | >1个百分点 | 减持或置换至更高股息标的 |
| 股价下跌 | >20%且基本面未变 | 增持（网格加仓） |
| 估值修复 | 达合理区间上沿（FCF倍数>8倍） | 减仓兑现 |
| 涨幅透支股息 | 涨幅>5年股息总和 | 清仓退出 |
| 组合指标偏离 | 平均FCF倍数>5倍 或 净现金比<30% | 整体调整 |

**再平衡频率**：每月检查一次（5分钟），核算组合平均FCF/市值、净现金/市值，识别减仓和加仓候选。

### 四、持仓状态标签体系（5级）

每只持仓标的必须标注当前状态标签，指导后续操作：

| 标签 | 代号 | 定义 | 后续操作建议 |
|------|------|------|-------------|
| **正常持仓** | G | 符合买入逻辑，未达卖出条件 | 持有观望，定期复核 |
| **已回本** | B | 前期被套，当前已回本或微利 | 评估是否切换至更优标的 |
| **高位持仓** | Y | 入场后成本较高，潜在回调风险 | 警惕回落，设置止盈/减仓线 |
| **关注未买入** | O | 符合估值逻辑，等待回调 | 设定买入触发价，耐心等待 |
| **历史遗留** | R | 非当前策略买入，需逐步清理 | 择机退出，不追加投入 |

**标签动态转换规则**：

```
O(关注未买入) → G(正常持仓)
  触发：价格达到买点，完成建仓

G(正常持仓) → Y(高位持仓)
  触发：股价从买入点上涨>50%，估值优势减弱

Y(高位持仓) → B(已回本)（若被套后回升）
  触发：价格下跌后又回升至成本价

G/Y/B → R(历史遗留)
  触发：策略调整，该标的不再符合新要求

任何状态 → 清仓
  触发：估值修复完成 / 基本面恶化 / 发现更优标的
```

### 五、组合韧性评估

量化评估组合抗风险能力和修复能力，月度核算：

| 指标 | 计算方法 | 优秀标准 | 警戒阈值 |
|------|---------|---------|---------|
| **正收益月占比** | 近12月盈利月数 / 12 | >75% | <50% |
| **最大回撤修复天数** | 从回撤低点回到前高所需天数 | <30天 | >60天 |
| **相对收益稳定性** | 连续跑赢基准月数 | >6个月 | 连续3月跑输 |
| **波动率** | 月度收益标准差 | <指数波动率80% | >指数波动率 |
| **组合平均FCF倍数** | 总(市值-净现金) / 总FCF | <5倍 | >8倍 |

**组合层面硬指标（三层安全垫）**：

| 指标 | 目标值 | 计算方式 | 低于目标时操作 |
|------|--------|---------|---------------|
| 平均剔除净现金FCF倍数 | < 5倍 | 组合总EV / 组合总FCF | 减仓高倍数标的，加仓低倍数标的 |
| 穿透回报 | >4%/年 | 组合总(股息+回购) / 组合总市值 | 减仓低派息标的 |
| 平均净现金/市值 | >30% | 组合总净现金 / 组合总市值 | 减仓净现金不足标的 |

**月度归因分析模板**：

```
### [月份]组合韧性评估

市场表现：
- 本月收益：[X%]
- 基准对比：跑赢/跑输 [基准指数] [Y%]
- 正收益月数：近12月 [N]/12 ([百分比]%)

归因分析：
1. 个股正向贡献：[标的A贡献X%]
2. 个股负向贡献：[标的B拖累Y%]
3. 行业轮动影响：[消费/周期/金融板块影响]

韧性验证：
- 连续3个月回撤？[是/否]
- 跑输基准>5%？[是/否]
- 最大回撤<15%？[是/否]

下月关注点：
- [高位持仓标的回调风险]
- [宏观事件对特定标的的影响]
```

### 六、绝对估值原则与网状思维

#### 6.1 网状思维 vs 点状思维

不押注单一标的未来，押注组合整体的估值修复：

| 维度 | 点状思维（错误） | 网状思维（正确） |
|------|----------------|----------------|
| 关注对象 | 单只个股的未来成长 | 组合整体的估值修复 |
| 买入逻辑 | "这家公司很好，长期持有" | "组合需要这个指标，低估值买入" |
| 卖出逻辑 | "基本面恶化才卖" | "估值修复完成即卖出，切换至更低估标的" |
| 估值标准 | "消费股可以给高估值"（行业特殊） | "绝对估值，不看行业，只看数字" |
| 持仓管理 | 买入不动，坐过山车 | "高低切换，动态再平衡" |
| 风险分散 | 假分散（买很多只但逻辑相同） | 真分散（组合层面指标硬约束） |

#### 6.2 绝对估值铁律

```
禁止：
├── "消费股可以给20倍PE" -> 错误
├── "科技股可以给高估值" -> 错误
├── "物业股只能给10倍PE" -> 错误
└── 任何"行业特殊估值" -> 都错误

允许：
├── 剔除净现金FCF倍数 < 5倍 -> 统一标准
├── FCF计算规范（股息不扣、CAPEX合理、防双标）-> 统一标准
├── 剔除净现金PE < 12 -> 统一标准
├── FCF/市值 > 10% -> 统一标准
└── 只看数字，不看行业 -> 绝对估值
```

### 七、宏观事件持仓联动评估

建立宏观事件与持仓的系统化联动评估机制，避免情绪化操作。

#### 7.1 评估框架

```
### 宏观事件：[事件名称]

影响标的：[股票名称]

影响分析：
| 维度 | 评估 | 逻辑链 |
|------|------|--------|
| 短期影响 | 压制/利好/重大风险 | [传导逻辑链] |
| 长期影响 | 趋势不变/假设破坏 | [核心假设是否受损] |
| 影响幅度 | 预估[-X%, +Y%] | [量化估算] |

应对策略：
- 若短期下跌>10% -> 评估加仓（基本面未变时）
- 若影响持续>6个月 -> 重新评估行业周期位置
- 若涉及核心假设破坏 -> 止损退出
```

#### 7.2 应对决策矩阵

| 事件性质 | 短期判断 | 长期判断 | 行动 |
|---------|---------|---------|------|
| 短期冲击 + 长期趋势不变 | 压制 | 不影响 | 持有不动，下跌>10%评估加仓 |
| 短期冲击 + 长期不确定 | 压制 | 待观察 | 暂停加仓，持续6个月后重估 |
| 核心假设被破坏 | 重大风险 | 假设失效 | 止损退出 |
| 短期利好 + 长期中性 | 提振 | 不影响 | 不追涨，维持纪律 |

#### 7.3 监控要点

```
每月检查（2分钟）：
├── 当前宏观事件清单：[列出]
├── 各事件对持仓标的的影响更新
├── 是否有新的宏观事件需要评估
└── 是否有事件影响持续超过6个月 -> 触发重估
```
